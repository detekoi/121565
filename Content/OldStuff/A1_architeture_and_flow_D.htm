Visual Testing

When implementing Visual testing using a test program drives an Application Under Test (AUT) through a series of check points by simulating user mouse and keyboard input. At each check point an image of the application window is captured. The first time a test is run the images are stored as a series of "baseline" images. On subsequent runs the newly captured images are compared to the baseline images and any differences are noted. The tester then uses the tool to view the differences and identify bugs.
