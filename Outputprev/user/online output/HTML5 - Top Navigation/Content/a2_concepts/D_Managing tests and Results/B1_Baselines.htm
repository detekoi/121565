Baselines

Baseline Names and IDs

Every test has an associated baseline. The baseline has a unique name and identifier. The name is composed of a numer of parts, some are provided by the test programer and others are defined be default by Eyes.

The test programer typically provides:

  • The Application Name : A string that may be common to a number of test, but unique for each group of tests.
  • The Test Name :  A string that must be unique for each test e.g. "Login Page"
  • The Viewport : the size of the window in which the application shoud run e.g. 800x599

Because the same set of test can be run under different platforms, and in each case the image content may be different, eyes automatically adds to the baseline the following:.

  • The platform operating system e.g. Windows 10, Mac OS X 10.11, Android 5.1
  • The Application Platform e.g. IE, Firefox, Chrome

If necessary, these may be overridden by the test code using the Eyes SDK

The viewport

The Viewport is the pixel dimensions on the screen of the area being matched. For example, on a browser, this is the visible area of the webpage within the browser window, and it doesn't include the browser buttons and tool bars. On other platforms such as mobile or desktop applications the viewport could include all of the content on the screen or all of the operating system window.

Apart from storing the viewport size as part of the baseline information, Eyes, also sets the viewport size to the correct dimension before running the test (on platforms where this is supported e.g. web browsers)

Additional Notes

  1. Selecting an appropriate viewport size is very important:
    1. When testing an application with a responsive interface, the test should be run multiple times, each time with a different viewport size to represent for example different target mobile device screen dimensions.
    2. The viewport should always be smaller than the screen resolution of the smallest screen it will be tested on. In the case of a web application, that runs on a browser, the viewport size should be small enough that when the browser's buttons and tool bars and the window frame are added the entire browser window will fit on the screen.
  2. The content of the page being displayed is not always visible in the viewport. There page may be scrollable, or it may contain scrollable elements.
